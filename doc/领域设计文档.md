## 1. 领域概述

### 1.1 业务背景
电商商品管理系统是电商平台的核心系统，负责管理商品全生命周期，包括商品创建、SKU管理、库存管理、分类管理、品牌管理、供应商管理等核心业务能力。

### 1.2 核心业务能力
1. **商品管理**: 商品创建、编辑、上下架、查询
2. **SKU管理**: 多维度SKU生成、属性管理、规格筛选
3. **库存管理**: 多仓库存、实时同步、库存预警、锁仓
4. **分类管理**: 无限级分类、属性模板
5. **品牌管理**: 品牌信息维护
6. **供应商管理**: 供应商信息、商品关联
7. **批量操作**: 批量导入、批量调整
8. **预警机制**: 库存预警、异常预警

## 2. 限界上下文划分 (Bounded Context)

### 2.1 限界上下文划分原则
根据PRD需求，按照业务职责单一性原则，将系统划分为以下限界上下文：

### 2.2 限界上下文详细说明

#### 2.2.1 商品上下文 (Product Context)
- **职责**: 管理商品基础信息、生命周期、上下架状态
- **核心领域模型**: Product（聚合根）
- **领域事件**: ProductCreated, ProductUpdated, ProductStatusChanged
- **防腐层**: 通过领域服务与SKU、分类、品牌上下文交互

#### 2.2.2 SKU上下文 (SKU Context)
- **职责**: 管理SKU生成、属性组合、规格管理
- **核心领域模型**: Sku（聚合根）、Attribute（实体）、AttributeValue（值对象）
- **领域事件**: SkuCreated, SkuAttributeChanged
- **防腐层**: 通过领域服务与商品、库存上下文交互

#### 2.2.3 库存上下文 (Inventory Context)
- **职责**: 管理多仓库存、库存同步、库存预警、锁仓
- **核心领域模型**: Inventory（聚合根）、Warehouse（实体）、StockLock（实体）
- **领域事件**: InventoryChanged, StockLocked, StockReleased, LowStockAlert
- **防腐层**: 通过领域服务与SKU、预警上下文交互

#### 2.2.4 分类上下文 (Category Context)
- **职责**: 管理商品分类树、属性模板
- **核心领域模型**: Category（聚合根）、AttributeTemplate（实体）
- **领域事件**: CategoryCreated, CategoryUpdated

#### 2.2.5 品牌上下文 (Brand Context)
- **职责**: 管理品牌信息
- **核心领域模型**: Brand（聚合根）
- **领域事件**: BrandCreated, BrandUpdated

#### 2.2.6 供应商上下文 (Supplier Context)
- **职责**: 管理供应商信息、供应商商品关联
- **核心领域模型**: Supplier（聚合根）、SupplierProduct（实体）
- **领域事件**: SupplierCreated, SupplierProductLinked

#### 2.2.7 预警上下文 (Alert Context)
- **职责**: 管理库存预警、异常预警
- **核心领域模型**: AlertRule（聚合根）、AlertRecord（实体）
- **领域事件**: AlertTriggered

#### 2.2.8 批量导入上下文 (Batch Import Context)
- **职责**: 管理批量导入任务
- **核心领域模型**: ImportTask（聚合根）
- **领域事件**: ImportTaskCreated, ImportTaskCompleted

## 3. 领域模型设计

### 3.1 通用语言 (Ubiquitous Language)

| 术语 | 英文 | 定义 |
|------|------|------|
| 商品 | Product | 描述商品基础信息，一个商品可以包含多个SKU |
| SKU | Stock Keeping Unit | 库存量单位，具有唯一编码，包含具体规格属性 |
| SPU | Standard Product Unit | 标准化产品单元，描述产品特性，如iPhone 15 |
| 属性 | Attribute | 描述商品特征，如颜色、尺寸 |
| 属性值 | Attribute Value | 属性的具体取值，如红色、XL |
| 库存 | Inventory | 特定SKU在特定仓库的库存数量 |
| 仓库 | Warehouse | 存储商品的物理场所 |
| 分类 | Category | 商品分类，支持无限级 |
| 品牌 | Brand | 商品品牌信息 |
| 供应商 | Supplier | 商品供应方 |
| 锁仓 | Stock Lock | 订单创建时预占库存 |

### 3.2 聚合根设计

#### 3.2.1 Product（商品聚合根）

**Product**
- productId: ProductId (唯一标识)
- name: String (商品名称)
- categoryId: CategoryId (分类ID)
- brandId: BrandId (品牌ID)
- supplierId: SupplierId (供应商ID)
- status: ProductStatus (状态: 草稿/上架/下架)
- mainImage: String (主图)
- price: Money (售价)
- marketPrice: Money (市场价)
- description: ProductDescription (图文描述)
- images: List<ProductImage> (图片列表)
- createTime: LocalDateTime
- updateTime: LocalDateTime

**业务规则**:
1. 商品名称在同一分类下不能重复
2. 上架商品必须包含至少一个SKU
3. 商品下架后不能购买
4. 商品删除前必须下架

#### 3.2.2 Sku（SKU聚合根）

**Sku**
- skuId: SkuId (唯一标识)
- productId: ProductId (所属商品ID)
- skuCode: String (SKU编码)
- barcode: Barcode (条形码)
- attributes: List<SkuAttribute> (属性组合)
- price: Money (售价)
- status: SkuStatus (状态)
- createTime: LocalDateTime

**SkuAttribute**
- attributeId: AttributeId (属性ID)
- attributeName: String (属性名称)
- attributeValue: AttributeValue (属性值)

**业务规则**:
1. 同一商品下SKU属性组合不能重复
2. SKU编码必须唯一
3. SKU数量不能超过限制(默认100)
4. SKU必须关联有效商品

#### 3.2.3 Inventory（库存聚合根）

**Inventory**
- inventoryId: InventoryId (唯一标识)
- skuId: SkuId (SKU ID)
- warehouseId: WarehouseId (仓库ID)
- availableStock: Quantity (可用库存)
- reservedStock: Quantity (预留库存)
- lockedStock: Quantity (锁定库存)
- warningThreshold: Quantity (预警阈值)
- version: Long (乐观锁版本号)
- updateTime: LocalDateTime

**StockLock**
- lockId: LockId (唯一标识)
- orderId: OrderId (订单ID)
- skuId: SkuId (SKU ID)
- warehouseId: WarehouseId (仓库ID)
- quantity: Quantity (锁定数量)
- status: LockStatus (锁定状态)
- expireTime: LocalDateTime (过期时间)

**业务规则**:
1. 可用库存 = 实际库存 - 锁定库存
2. 锁定库存不能超过可用库存
3. 库存不能为负数
4. 锁仓有过期时间，过期自动释放

#### 3.2.4 Category（分类聚合根）

**Category**
- categoryId: CategoryId (唯一标识)
- name: String (分类名称)
- parentId: CategoryId (父分类ID)
- level: Integer (层级)
- sortOrder: Integer (排序)
- icon: String (图标)
- attributeTemplates: List<AttributeTemplate> (属性模板)
- status: CategoryStatus (状态)
- createTime: LocalDateTime

**AttributeTemplate**
- templateId: TemplateId (唯一标识)
- attributeName: String (属性名称)
- attributeType: AttributeType (类型: 文本/数值/枚举)
- required: Boolean (是否必填)
- options: List<String> (选项值)
- sortOrder: Integer (排序)

**业务规则**:
1. 分类名称在同一层级下不能重复
2. 有子分类时不能删除
3. 分类层级不能超过5级
